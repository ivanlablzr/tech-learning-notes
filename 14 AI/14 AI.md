## The five views

**Bridge topic it owns — GPU/CUDA** (massively parallel linear algebra for training/inference; memory bandwidth is the real bottleneck), plus **vector DBs ↔ retrieval** (RAG links [[08 Databases]] to LLM apps) and **distributed training** (reuses [[12 Distributed Systems]]).

**Learning path:** ML foundations → classic models → optimization (gradient descent) → deep learning (backprop, CNN/RNN) → Transformers & LLMs → applied LLMs (prompting, RAG, agents) → AI infrastructure (GPUs, vector DBs, serving).

**Projects:** a neural net from scratch (backprop by hand) → **RAG over this vault** → fine-tune a small model → a vector-search service. See [[Learning Paths & Projects#Project Roadmap]].

## AI in one page

**History:** Turing test (1950) → "AI" coined (1956) → expert systems → AI winters → ML revival (Deep Blue 1997) → **deep learning breakthrough (2012)** → **Transformer (2017)** → ChatGPT mainstreams LLMs (2022) → **agentic AI** (acts, not just answers).

**How it's built:** [[Machine Learning & Deep Learning|ML]] (supervised/unsupervised/reinforcement) → **deep learning** (many-layer neural nets) → the **Transformer** (attention) powering LLMs; image generators use **diffusion**; **MoE** activates only a fraction of a huge model per token for efficiency. The lifecycle is **training** (once, $10M–500M) → **fine-tuning** (occasional; LoRA/QLoRA) → **inference** (every message).

**Key terms:** *parameters* (the "B" in 70B), *tokens* (~¾ word), *context window*, *embeddings* (meaning as vectors), *temperature* (randomness), *quantization* (smaller weights), *hallucination*, *RAG* (ground answers in your documents), *RLHF* (align to human preference), *open weights* vs *closed*, *MCP* (tool-connection standard). Benchmarks: SWE-bench (real GitHub issues), MMLU (academic knowledge), GPQA (expert science).

## Practical AI use & safety

**Cloud vs local:** cloud models (Claude, GPT, Gemini) are most capable but your data leaves the machine; **local** models (via Ollama — Llama/Qwen/Gemma) keep data private and run offline, bounded by VRAM. For sensitive/work material, prefer local models or paste manually into a browser chat; disable training-data retention in account settings.

> **Security principle:** treat any AI with system access *like a smart but unscrupulous intern* — least privilege, sandbox, audit.

The four real risks of agentic/coding AI:
1. **Prompt injection** — hidden instructions in a repo/file/web page hijack the agent. *Never point an active agent at untrusted code; use read-only/sandbox mode.*
2. **Extended system access** — it can read/modify/delete files and run commands. *Run as a non-admin user; review the plan before approving.*
3. **Code/data exfiltration** — cloud processing sends your code off-machine. *Use local models for work infrastructure.*
4. **Vendor dependency** — keep a local fallback so one provider's outage/price change doesn't strand you.

Protect API keys (env var → OS keychain, never hard-coded), `.gitignore` secrets, verify suggested packages, and never paste credentials into a chat.

**Prompting** (full techniques in [[LLMs & Prompting]]): be specific; give a role; show examples; "think step by step"; use XML tags; iterate. **Anti-sycophancy** matters — RLHF makes models agreeable, so explicitly ask *"what's wrong with my thinking?"*, *"argue against this"*, *"rate this honestly 1–10."*

**Ethics & impact:** training data carries **bias** (worse facial recognition on darker skin, biased hiring); the **black-box** problem makes decisions hard to explain (→ interpretability research); principles = beneficence, non-maleficence, autonomy, justice, explicability. A subtler risk is **psychological**: a sycophantic AI can reinforce false beliefs through validation ("bidirectional belief amplification") — use AI as one input among many, not a replacement for your own reasoning or human relationships. The deepest long-term concern is **alignment** — keeping increasingly capable systems controllable and aligned with human values (see [[AI Foundations]]).
