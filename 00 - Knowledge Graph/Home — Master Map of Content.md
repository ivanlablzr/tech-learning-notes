This vault is a **knowledge graph for understanding modern computing from first principles** — from electrons to global platforms. It is optimized for systems thinking and relationships, not note quantity.

> [!quote] Guiding principle
> For every concept, answer: **(1)** Why does it exist? **(2)** What problem does it solve? **(3)** What depends on it? **(4)** What does it depend on? **(5)** How does it interact with the rest of the system?

## The convergence

The domains are not separate trees. Two paths — one rising from hardware, one descending from software — meet at the same place:

```text
Physics → Electronics → Hardware → OS → Networking → Security → Cloud → DevOps → Platform Engineering
Math → Programming → Applications → Databases → Distributed Systems → Cloud Native → Platform Engineering
```

Where they cross are the **bridge topics** (TCP/IP, DNS, TLS, IAM, containers, Kubernetes, hypervisors, observability). Those are the highest-value notes in the vault. See [[Knowledge Graph#Bridge Topic Map]].

## How to use this vault

1. Pick a track in [[Learning Paths & Projects#Learning Paths]].
2. Before creating any note, apply the test in [[Analysis & Gaps#Foundation Analysis]]: *can I explain why this topic exists?*
3. Tie every cluster of notes to a project from [[Learning Paths & Projects#Project Roadmap]].
4. Keep filling the graph — links matter more than folders.
